# Relation between Dockerfile and Docker Compose in RIAD_SERVER

The goal is to explain the relationship between the `Dockerfile` and `docker-compose.yml` in the project and implement a new web server service in a dedicated directory.

## Analysis
- `RIAD_SERVER/Dockerfile`: Defines the build process for the Go server application using a multi-stage build.
- `docker-compose.yml`: Orchestrates the `server` and `postgres` services. The `server` service is configured to build its image from the `RIAD_SERVER/Dockerfile`.

## Implementation: Adding `RIAD_web` (Golang Version)
The user wants to create a `RIAD_web` directory similar to `RIAD_SERVER`, but implemented in Golang instead of using Nginx.

### Steps:
1. **Create `RIAD_web` Directory**: 
   - Initialize `/home/faical/Projects/RIAD_PROJECT/RIAD_web`.
2. **Create `RIAD_web/main.go`**:
   - Implement a simple Go web server that serves the frontend assets (from `../RIAD_APP/frontend/dist`) or provides a web interface.
3. **Create `RIAD_web/Dockerfile`**:
   - Implement a multi-stage build similar to `RIAD_SERVER/Dockerfile`:
     - **Stage 1 (Build)**: Use `golang:alpine` to compile the Go binary.
     - **Stage 2 (Runtime)**: Use `alpine:latest` to run the binary.
4. **Update `docker-compose.yml`**:
   - Add the `web` service.
   - Set `build: { context: ./RIAD_web, dockerfile: Dockerfile }`.
   - Expose port `80:8080` (or similar).
   - Add `depends_on: [server]` to ensure backend availability.
